# FlowMind 系统 UI 交互说明

## 1. 概述

本文档详细描述FlowMind审批系统的用户界面交互设计，包括主要功能模块的界面布局、用户操作流程、交互细节和设计规范。FlowMind系统基于RuoYi-Cloud微服务框架，集成Flowable工作流引擎，实现高效的审批流程管理。本文档旨在指导UI/UX设计师、前端开发人员和测试人员理解系统的交互逻辑，确保最终实现的用户界面符合预期的交互体验。

## 2. 界面架构

### 2.1 整体布局

FlowMind系统采用现代化的左侧导航 + 顶部工具栏 + 主内容区的布局模式：

```
+-----------------------------------------------+
| 顶部导航栏（Logo、全局搜索、通知、用户信息）      |
+----------+-------------------------------------+
|          |                                     |
| 左侧菜单  |             主内容区                 |
|          |                                     |
|（可折叠） |                                     |
|          |                                     |
+----------+-------------------------------------+
```

### 2.2 响应式设计

- **桌面端**（≥1200px）：完整三栏布局，左侧菜单固定展开
- **平板端**（768px-1199px）：左侧菜单可折叠，右侧面板在必要时隐藏
- **移动端**（<768px）：左侧菜单折叠为汉堡菜单，采用单栏布局

## 3. 主要功能模块交互流程

### 3.1 登录流程

**目标**：用户通过身份验证进入系统

**交互步骤**：
1. 用户访问系统URL，显示登录页面
2. 用户输入用户名和密码
3. 点击「登录」按钮或按Enter键
4. 系统验证凭据：
   - 验证成功：显示主界面，同时显示欢迎消息
   - 验证失败：显示错误提示，清空密码输入框，保留用户名

**特殊情况**：
- 记住密码：勾选后保存登录状态7天
- 忘记密码：点击链接跳转到密码重置流程
- 多因素认证：根据配置可能需要短信验证码或其他二次验证

### 3.2 申请提交流程

**目标**：用户提交新的审批申请

**交互步骤**：
1. 从左侧菜单选择「工作流」→「发起流程」
2. 选择需要发起的流程定义
3. 系统加载相应的申请表单
4. 用户填写申请表单：
   - 必填项用红色星号标记
   - 实时验证字段格式
   - 提供表单填写提示
5. 上传相关附件：
   - 点击上传区域或拖拽文件
   - 显示上传进度和已上传文件列表
   - 支持删除已上传文件
6. 点击「提交」按钮
7. 系统进行表单验证：
   - 验证通过：显示提交成功提示，跳转至申请详情页
   - 验证失败：在相应字段下方显示错误信息

**辅助功能**：
- 表单草稿保存、编辑和管理
- 历史申请内容查看

### 3.3 审批处理流程

**目标**：审批人处理待审批的申请

**交互步骤**：
1. 登录系统后，待审批任务在首页显示或通过顶部通知提醒
2. 点击待审批任务或从左侧菜单进入「工作流」→「待办任务」
3. 在待审批列表中选择一个申请查看详情
4. 申请详情页展示：
   - 申请基本信息
   - 相关附件
   - 审批历史记录
5. 审批人审核申请内容
6. 做出审批决定：
   - 点击「通过」按钮：显示意见输入框
   - 点击「驳回」按钮：显示驳回原因输入框
   - 点击「退回」按钮：显示退回原因输入框
   - 点击「撤销」按钮：取消已提交的申请
7. 输入相关意见，点击「确认」按钮
8. 系统更新申请状态，显示操作成功提示，自动刷新待审批列表

**辅助功能**：
- 审批意见模板
- 审批超时提醒

### 3.4 申请查询与管理流程

**目标**：用户查看和管理自己的申请

**交互步骤**：
1. 从左侧菜单选择「工作流」→「我的流程」
2. 显示申请列表，默认显示所有申请
3. 使用筛选和搜索功能查找特定申请：
   - 按状态筛选（全部、待审批、审批中、已通过、已驳回、已撤销）
   - 按时间范围筛选
   - 关键字搜索（标题、申请人等）
4. 在列表中点击申请标题查看详情
5. 在申请详情页，根据申请状态可执行不同操作：
   - 待审批状态：可进行修改或撤销操作
   - 已审批状态：仅可查看详情，无法修改
6. 修改申请时，表单预填当前信息，修改后重新提交
7. 撤销申请时，需要选择撤销原因

**辅助功能**：
- 申请数据导出（Excel）
- 申请状态变更通知
- 申请进度可视化展示

### 3.5 流程监控与管理流程

**目标**：管理员监控和管理系统中的审批流程

**交互步骤**：
1. 管理员从左侧菜单选择「工作流」→「流程管理」
2. 显示所有配置的流程列表
3. 点击流程名称查看流程详情或编辑流程
4. 流程编辑界面提供可视化设计器：
   - 拖拽式流程节点添加
   - 连接线设置流转条件
   - 节点属性配置（审批人、条件等）
5. 修改完成后保存流程配置
6. 可以启用/禁用流程
7. 从「工作流」→「流程实例」查看正在运行的流程实例

**辅助功能**：
- 流程模板保存和复用
- 流程运行日志查询
- 异常流程手动干预

## 4. 界面组件交互设计

### 4.1 表格组件

**功能**：用于展示申请列表、审批历史等数据

**交互细节**：
- **排序**：点击表头可按该列升序/降序排序
- **筛选**：表头提供筛选下拉菜单，可选择筛选条件
- **分页**：底部提供分页控件，显示总数、每页条数选择、页码导航
- **复选框**：支持单行选择和批量选择，批量选择后显示批量操作按钮
- **列宽调整**：可拖动列边界调整宽度
- **列显示隐藏**：可配置显示哪些列
- **行悬停**：鼠标悬停在行上时有高亮效果
- **操作菜单**：每行提供操作按钮或下拉菜单，包含查看、编辑、删除等操作

### 4.2 表单组件

**功能**：用于数据录入和编辑

**交互细节**：
- **输入验证**：实时验证输入格式，错误时提供红色边框和错误提示
- **必填项标记**：必填字段用红色星号标记
- **输入提示**：提供输入格式和要求的帮助提示
- **自动填充**：常用字段支持自动填充功能
- **日期选择器**：点击弹出日历选择器
- **文件上传**：拖拽或点击上传，显示上传进度
- **级联选择**：根据前面的选择动态加载后面的选项
- **表单保存**：自动保存草稿，防止数据丢失

### 4.3 弹窗组件

**功能**：用于确认操作、显示详情、表单编辑等

**交互细节**：
- **打开方式**：点击按钮或链接触发
- **关闭方式**：点击关闭按钮、右上角X、取消按钮或点击外部区域
- **大小调整**：支持拖动边缘调整大小
- **位置调整**：支持拖动标题栏移动位置
- **加载状态**：加载内容时显示加载动画
- **确认对话框**：重要操作需要二次确认，提供确认和取消按钮

### 4.4 通知组件

**功能**：向用户推送系统消息和提醒

**交互细节**：
- **通知中心**：顶部导航栏显示通知图标和未读数量
- **消息类型**：区分普通消息、重要通知、警告信息等不同样式
- **消息显示**：新消息出现时从右上角滑入，一段时间后自动消失
- **交互操作**：点击消息可查看详情，提供标记已读、全部已读功能
- **通知偏好**：用户可配置接收哪些类型的通知

### 4.5 导航组件

**功能**：系统功能和页面导航

**交互细节**：
- **左侧菜单**：支持多级菜单，当前选中项高亮显示
- **菜单折叠**：点击折叠按钮收起/展开菜单
- **顶部导航**：包含Logo、全局搜索、通知、用户头像等
- **面包屑导航**：显示当前页面路径，支持快速跳转
- **标签页导航**：支持多标签页操作，可关闭不需要的标签

## 5. 特殊交互场景

### 5.1 异常情况处理

**网络异常**：
- 检测到网络异常时显示离线提示
- 自动尝试重连
- 关键操作支持离线缓存，网络恢复后自动提交

**数据加载超时**：
- 设置合理的加载超时时间
- 超时后显示错误提示和重试按钮
- 提供取消加载选项

**权限不足**：
- 尝试访问无权限功能时显示友好的权限不足提示
- 提供申请权限或联系管理员的选项

### 5.2 键盘快捷操作

系统支持以下键盘快捷操作，提高操作效率：

| 快捷键 | 功能描述 |
|--------|----------|
| Ctrl + F | 打开全局搜索 |
| Ctrl + N | 新建申请 |
| Ctrl + S | 保存表单 |
| Esc | 关闭弹窗或取消操作 |
| Enter | 提交表单或确认操作 |
| Tab | 表单字段间导航 |

## 6. 无障碍设计

### 6.1 键盘操作支持

- 所有功能可通过键盘访问和操作
- 清晰的键盘焦点指示
- 支持常见键盘快捷键

### 6.2 屏幕阅读器支持

- 所有重要内容可被屏幕阅读器正确识别
- 提供适当的ARIA标签
- 表单元素有关联的标签

### 6.3 色彩与对比度

- 文字与背景的对比度符合WCAG AA标准
- 不依赖颜色作为唯一的信息传达方式
- 提供高对比度模式选项

## 7. 设计规范

### 7.1 颜色方案

**主色调**：#1890ff（蓝色）- 代表专业和信任
**辅助色**：
- 成功：#52c41a（绿色）
- 警告：#faad14（橙色）
- 错误：#f5222d（红色）
- 信息：#1890ff（蓝色）

**中性色**：
- 背景：#f0f2f5（浅灰色）
- 卡片背景：#ffffff（白色）
- 文字：#262626（深灰色）
- 次要文字：#8c8c8c（灰色）

### 7.2 排版规范

**字体**：
- 主要字体：-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif
- 代码字体：'SFMono-Regular', Consolas, 'Liberation Mono', Menlo, Courier, monospace

**字号规范**：
- 页面标题：18-24px
- 区域标题：16-18px
- 正文文字：14px
- 辅助文字：12px

### 7.3 间距规范

- 组件间距：16px
- 区域间距：24px
- 行间距：1.5倍字号
- 段落间距：16px
