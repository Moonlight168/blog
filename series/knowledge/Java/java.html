什么是面向对和面向过程，两者间的区别是什么？

回答：

面向过程是一种以功能为中心的编程方式，通过一步步的操作来完成任务；

面向对象是一种以对象为中心的编程方式，将数据和行为封装到对象中，通过对象之间的协作完成任务。

区别：

  • 面向过程关注“过程”，面向对象关注“对象”
  • 面向过程通过函数组织程序，面向对象通过类和对象组织程序
  • 面向过程数据暴露，面向对象数据封装
  • 面向过程适合小型程序，面向对象适合复杂系统
  • 面向过程可读性较差，面向对象更易维护和扩展

Java 中的类加载机制？

回答： Java 中的类加载机制是指类从被加载到内存再到实例化的整个过程，分为以下几个阶段：

  1. 加载（Loading）：通过类的全限定名加载 .class 文件到内存，生成 Class 对象。

  2. 连接（Linking）：包括：

    • 验证：校验字节码是否合法；
    • 准备：为静态变量分配内存并初始化默认值；
    • 解析：将常量池中的符号引用转为直接引用。

  3. 初始化（Initialization）：执行类的 <clinit> 方法，即静态变量赋初值和静态代码块执行。

类加载器有三种主要类型：

  • 启动类加载器（Bootstrap ClassLoader）：加载核心类库，如 rt.jar
  • 扩展类加载器（ExtClassLoader）：加载 jre/lib/ext 目录下的类
  • 应用类加载器（AppClassLoader）：加载用户类路径下的类（如项目代码）

类加载器采用双亲委派模型：先由父类加载器尝试加载，只有父类加载器无法加载时，子类加载器才尝试加载，确保核心类不会被重复加载或被篡改。

堆和栈在 Java 中的应用有哪些？

回答：

  • 栈（Stack）：用于存储方法调用过程中的局部变量（如基本类型、对象引用），每个线程有独立的栈，具有先进后出的特点，方法执行完自动出栈，释放内存。

  • 堆（Heap）：用于存储对象实例和数组，是所有线程共享的内存区域，生命周期较长，由垃圾回收机制管理。所有使用 new 创建的对象都分配在堆中。

什么是自动装箱与拆箱？原理是什么？为什么要这样做？

回答：

  • 自动装箱：基本类型自动转换为对应的包装类对象，例如：int → Integer
  • 自动拆箱：包装类对象自动转换为对应的基本类型，例如：Integer → int

原理： 由编译器在编译阶段自动插入代码，将基本类型与包装类之间进行 valueOf() 和 xxxValue() 方法的转换。

例如：

Integer a = 10;        // 自动装箱：Integer.valueOf(10)  
int b = a;             // 自动拆箱：a.intValue()

原因： Java 为了简化基本类型与对象类型之间的转换，使它们能在集合类（如 List、Map）中灵活使用，同时提升代码可读性与开发效率。

为什么在 List 和 Map 中要使用包装类？

回答：

因为 Java 的集合类（如 List、Map）只能存储对象，不能直接存储基本类型（如 int、double），所以必须使用对应的包装类（如 Integer、Double）来存储基本数据。

这是因为集合是基于泛型设计的，而泛型只支持引用类型，不支持基本类型。

例如：

List<int> list = new ArrayList<>(); // 错误  
List<Integer> list = new ArrayList<>(); // 正确，自动装箱

总结： 使用包装类是为了让基本类型能够在集合中使用，并支持泛型和面向对象特性。

Java SE vs Java EE

回答： Java SE 是 Java 的基础版本，Java EE 是 Java 的高级版本。Java SE 更适合开发桌面应用程序或简单的服务器应用程序，Java EE 更适合开发复杂的企业级应用程序或 Web 应用程序

如何解决浮点数运算的精度丢失问题？

BigDecimal 可以实现对浮点数的运算，不会造成精度丢失。通常情况下，大部分需要浮点数精确运算结果的业务场景（比如涉及到钱的场景）都是通过 BigDecimal 来做的。

为什么 float 或 double 会造成精度丢失？为什么 BigDecimal 不会？

回答：

  • float 和 double 会精度丢失，因为它们采用 二进制存储，某些十进制小数（如 0.1、0.2）在二进制中无法精确表示，只能近似存储，导致运算结果出现误差。

  • BigDecimal 不会精度丢失，因为它使用的是 任意精度的十进制表示，内部通过 字符串或整数+位数 精确存储小数，适合金融等对精度要求高的场景。

示例对比：

System.out.println(0.1 + 0.2); // 输出 0.30000000000000004
System.out.println(new BigDecimal("0.1").add(new BigDecimal("0.2"))); // 输出 0.3

总结：float/double 是近似值，适合科学计算；BigDecimal 是精确值，适合高精度商业计算。

什么是字节码? 采用字节码的好处是什么?

回答：

字节码是编译器将源代码（如 Java 代码）转换成的一种中间代码，它是与平台无关的二进制指令，能够被 Java 虚拟机（JVM）执行。Java 源代码首先被编译成字节码文件（.class 文件），然后通过 JVM 解释或即时编译（JIT）执行。

采用字节码的好处：

  1. 平台独立性：字节码是一种与平台无关的中间语言，只要有 JVM，就可以在任何操作系统或硬件平台上执行 Java 程序。

  2. 提高性能：字节码可以通过 JVM 进行优化（如即时编译技术 JIT），根据运行时环境进行动态优化，提升执行效率。

  3. 安全性：字节码被 JVM 执行时可以进行校验，避免恶意代码的执行，因此字节码提供了一定的安全性。

  4. 可移植性：同一份 Java 字节码文件可以在不同操作系统和硬件平台上运行，增强了程序的可移植性。

总结： 字节码使 Java 程序可以做到一次编写、处处运行，这是 Java 提供的“跨平台”的关键所在。

重载和重写有什么区别？

回答：

维度重载 (Overloading)重写 (Overriding)
定义同一个类中，方法名相同，但参数列表不同（数量或类型）子类重新定义父类的同名方法，方法签名相同
方法签名方法名相同，参数不同（类型、数量、顺序）方法名和参数列表完全相同
返回类型可以不同必须相同，或子类类型与父类方法的返回类型是继承关系
访问修饰符可以不同（但不能比父类方法更严格）不能缩小父类方法的访问权限
调用时机通过方法调用时根据传入参数决定具体调用哪个重载方法通过父类引用调用时，实际执行的是子类重写的方法
多态性无法实现多态可以实现多态
编译时/运行时编译时决定调用哪个方法（静态绑定）运行时决定调用哪个方法（动态绑定）

总结：

  • 重载：同名方法，参数不同，发生在同一个类中，用于增加方法的灵活性。
  • 重写：子类修改父类方法，方法签名完全相同，用于提供子类特定的实现，支持多态。

== 和 .equals() 有什么区别？

回答：== 比较的是对象的内存地址是否相同（引用是否相等）；

.equals() 比较的是对象的内容是否相同，通常会被重写以实现自定义比较逻辑。

⭐ Java 中的面向对象三大特性是什么？

回答：

Java 面向对象的三大特性是：

  1. 封装： 把数据和行为绑定在一起，对外隐藏内部实现，只暴露必要的接口，提升安全性和模块化。 关键词：private 属性 + public 方法。

  2. 继承： 子类自动拥有父类的属性和方法，实现代码复用。 关键词：extends，支持单继承。

  3. 多态： 同一接口，调用不同对象的实现，运行时动态绑定。 表现形式：方法重写 + 父类引用指向子类对象。

接口和抽象类的区别，抽象类的作用？

接口和抽象类的区别

  1. 定义不同
    • 接口（Interface）：是一种规范，定义了一组没有具体实现的方法，类通过实现接口来承诺实现这些方法。接口更注重行为的约束。
    • 抽象类（Abstract Class）：是一种不完全实现的类，可以包含抽象方法（没有方法体）和普通方法（有具体实现）。抽象类更注重代码的复用与继承结构。 。
  2. 继承和实现关系： 一个类可以实现多个接口（多继承），但只能继承一个抽象类。

抽象类主要用于：

代码复用：抽象类可以定义公共的成员变量和方法，子类继承后直接复用，减少重复代码。

强制规范：抽象类中的抽象方法定义了一种“必须实现”的行为，子类继承后必须实现这些方法，起到类似接口的约束作用。

模板设计：可以定义一套调用流程的骨架方法（模板方法模式），具体的细节由子类实现。

Java 中的 this 关键字

回答：

this 是 Java 中的一个引用当前对象自身的关键字，用于在类的内部访问当前对象的属性、方法或构造器。

注意事项：

  • this 只能在 非静态方法或构造器中使用； 因为 this 代表当前类的实例对象，而静态方法属于类本身，不依赖于任何对象。

Java 中的 static 关键字

回答：

static 是 Java 中的静态修饰符，可用于修饰变量、方法、代码块和内部类，表示它属于类而不是实例。

用法与含义：

  1. 静态变量（类变量） 所有对象共享，类加载时创建，内存中只有一份。

    static int count;

  2. 静态方法 不依赖实例，可通过类名直接调用；不能访问非静态成员。

    static void print() { ... }

  3. 静态代码块 在类加载时执行一次，用于初始化静态变量。

    static {
    // 初始化逻辑
}

  4. 静态内部类 不依赖外部类实例，可直接访问外部类的静态成员。

    static class Inner { ... }

注意事项：

  • 静态方法中不能使用 this 或 super；
  • 静态成员优先于对象存在；
  • 通常用于工具类（如 Math、Collections）或常量类。

深拷贝和浅拷贝的区别？

浅拷贝复制对象的引用，深拷贝复制对象的内容，即创建一个新对象，然后复制对象中的内容。

String、StringBuffer、StringBuilder 的区别？

String 是不可变类，StringBuffer 和 StringBuilder 是可变类，StringBuffer 是线程安全的， StringBuilder 是线程不安全的

throw 和 throws 的区别？

throw 用于抛出异常，throws 用于声明异常

什么是序列化和反序列化，为什么要序列化和反序列化，在Java中如何实现？

序列化： 将对象的状态转换为字节流的过程。

反序列化： 将字节流恢复为对象的过程。

条件： 类必须实现 Serializable 接口或者Jackson库序列化

不进行序列化后果：

  • 无法将对象保存到文件或数据库中
  • 无法在网络传输中传递对象，如 RPC（远程过程调用）中，客户端和服务器之间需要通过网络传输对象。
  • 在执行类似于对象持久化、远程通信、对象缓存等操作时，未序列化的对象将无法被处理，从而抛出运行时异常

Java中的反射机制？

回答：

  • 定义：反射是 Java 提供的一种在运行时动态获取类的信息并操作其成员（属性、方法、构造器）的方法

  • 作用：动态创建对象、调用方法、访问属性，常用于框架、工具类、注解处理等

  • 核心类：Class、Field、Method、Constructor（均在 java.lang.reflect 包中）

  • 示例：

    Class<?> clazz = Class.forName("com.example.User");
Object obj = clazz.getDeclaredConstructor().newInstance();
Method method = clazz.getMethod("getName");
method.invoke(obj);

  • 注意事项：反射会影响性能，且破坏封装性，使用时需谨慎。

Java中的代理模式

回答：

一、概念理解

代理模式（Proxy Pattern）：为其他对象提供一种代理以控制对这个对象的访问。

二、分类与区别

类型是否依赖接口实现方式特点/使用场景
静态代理是编写代理类手写繁琐，但结构清晰
动态代理（JDK）是反射+接口动态生成代理类，运行时增强
CGLIB 代理否ASM 字节码生成子类适用于无接口类，性能略高
Spring AOP 代理依赖 JDK/CGLIB运行时决定Spring 自动选择代理方式（默认先选 JDK）

三、静态代理

定义： 编译期由程序员手动创建代理类，代理类和目标类实现同一接口。

示例结构：

// 接口
public interface UserService {
    void addUser();
}

// 目标类
public class UserServiceImpl implements UserService {
    public void addUser() {
        System.out.println("添加用户");
    }
}

// 代理类
public class UserServiceProxy implements UserService {
    private UserService target = new UserServiceImpl();

    public void addUser() {
        System.out.println("权限校验");
        target.addUser();
        System.out.println("日志记录");
    }
}

四、JDK 动态代理（基于接口）

定义： 运行时生成代理对象，目标类必须实现接口。

特点： 基于接口+反射，灵活通用。

核心类： Proxy + InvocationHandler

UserService target = new UserServiceImpl();

UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    (proxyObj, method, args) -> {
        System.out.println("权限校验");
        Object result = method.invoke(target, args); // 执行原方法
        System.out.println("日志记录");
        return result;
    }
);

proxy.addUser(); // 调用代理对象方法

五、CGLIB 动态代理（基于继承）

CGLIB（Code Generation Library） 是一个强大的字节码增强工具，它通过继承目标类并重写方法来实现代理，不需要目标类实现接口。

特点： 基于继承+字节码生成，不能代理 final 类和方法。

核心类： Enhancer + MethodInterceptor

public class MyMethodInterceptor implements MethodInterceptor {
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("前置增强");
        Object result = proxy.invokeSuper(obj, args); // 调用目标方法
        System.out.println("后置增强");
        return result;
    }
}

// 使用
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(UserServiceImpl.class);
enhancer.setCallback(new MyMethodInterceptor());

UserServiceImpl proxy = (UserServiceImpl) enhancer.create();
proxy.addUser();

六、Spring AOP 与代理模式关系

Spring AOP 的本质就是使用了代理模式：

  • 如果目标类实现了接口 ➜ 使用 JDK 动态代理
  • 如果没有接口或强制使用类代理 ➜ 使用 CGLIB 动态代理

配置方式示例：

spring:
  aop:
    proxy-target-class: true  # 强制使用 CGLIB

七、代理模式的优缺点

优点：

  • 解耦核心业务与横切逻辑（如权限、事务、日志）
  • 可灵活控制对象访问，增强安全性、性能

缺点：

  • 多层代理可能导致维护困难
  • CGLIB 代理不能代理 final 类/方法,CGLIB 代理需要继承目标类生成子类和重写对应的方法
  • final 类的限制：在 Java 中，final 类不能被继承
  • final 方法不能被重写
