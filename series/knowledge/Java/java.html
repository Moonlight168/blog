面向对和面向过程的区别

回答：

面向过程编程（POP）：面向过程把解决问题的过程拆成一个个方法，通过一个个方法的执行解决问题。

面向对象编程（OOP）：面向对象会先抽象出对象，然后用对象执行方法的方式解决问题。

OOP 开发的程序一般具有下面这些优点：

  • 易维护：由于良好的结构和封装性，OOP 程序通常更容易维护。
  • 易复用：通过继承和多态，OOP 设计使得代码更具复用性，方便扩展功能。
  • 易扩展：模块化设计使得系统扩展变得更加容易和灵活。

Java 中的类加载机制？

回答： 在 Java 中，类加载机制是指在程序运行时，JVM 会根据类的字节码（.class 文件）来加载类，并创建类的实例。

Java 中的八大基本类型？

回答： Java 的八种基本类型包括：byte、short、int、long、float、double、char 和 boolean。

Java 中如何判断一个对象是基本类型还是引用类型？

回答： 在 Java 中，基本类型是值类型，它们直接存储在栈中，而引用类型是引用类型，它们存储在堆中，指向对象。

自动装箱与拆箱了解吗？原理是什么？为什么要这样做？

回答：

装箱：将基本类型用它们对应的引用类型包装起来；

拆箱：将包装类型转换为基本数据类型

原理： 本质上，自动装箱是编译器的语法糖，在编译阶段， Java 会自动在适当位置调用：

  • Integer.valueOf(int) → 装箱
  • intValue()、doubleValue() 等方法 → 拆箱
Integer i = 10;  //装箱
int n = i;   //拆箱

目的： 简化开发，在 JDK 1.5 之前，如果要把 int 放入 List<Integer>，需要手动装箱：

List<Integer> list = new ArrayList<>();
list.add(Integer.valueOf(1));  // 手动装箱
int x = list.get(0).intValue(); // 手动拆箱

Java SE vs Java EE

回答： Java SE 是 Java 的基础版本，Java EE 是 Java 的高级版本。Java SE 更适合开发桌面应用程序或简单的服务器应用程序，Java EE 更适合开发复杂的企业级应用程序或 Web 应用程序

如何解决浮点数运算的精度丢失问题？？

BigDecimal 可以实现对浮点数的运算，不会造成精度丢失。通常情况下，大部分需要浮点数精确运算结果的业务场景（比如涉及到钱的场景）都是通过 BigDecimal 来做的。

什么是字节码?采用字节码的好处是什么?

回答： 在 Java 中，JVM 可以理解的代码就叫做字节码（即扩展名为 .class 的文件），它不面向任何特定的处理器，只面向虚拟机。 Java 语言通过字节码的方式，在一定程度上解决了传统解释型语言执行效率低的问题， 同时又保留了解释型语言可移植的特点。所以，由于字节码并不针对一种特定的机器， 因此，Java 程序无须重新编译便可在多种不同操作系统的计算机上运行。

重载和重写有什么区别？

回答： 重载就是同样的一个方法能够根据输入数据的不同，做出不同的处理

重写就是当子类继承自父类的相同方法，输入数据一样，但要做出有别于父类的响应时，你就要覆盖父类方法

== 和 .equals() 有什么区别？

回答：== 比较的是对象的内存地址是否相同（引用是否相等）；.equals() 比较的是对象的内容是否相同，通常会被重写以实现自定义比较逻辑。

⭐ Java 中的面向对象三大特性是什么？

回答：

  • 封装：隐藏内部实现，提供统一接口；
  • 继承：子类复用父类的属性和方法；
  • 多态：相同方法调用表现出不同行为（重写、重载）。

接口和抽象类的区别，抽象类的作用？

接口和抽象类的区别

  1. 定义不同
    • 接口（Interface）：是一种规范，定义了一组没有具体实现的方法，类通过实现接口来承诺实现这些方法。接口更注重行为的约束。
    • 抽象类（Abstract Class）：是一种不完全实现的类，可以包含抽象方法（没有方法体）和普通方法（有具体实现）。抽象类更注重代码的复用与继承结构。 。
  2. 继承和实现关系： 一个类可以实现多个接口（多继承），但只能继承一个抽象类。

抽象类主要用于：

代码复用：抽象类可以定义公共的成员变量和方法，子类继承后直接复用，减少重复代码。

强制规范：抽象类中的抽象方法定义了一种“必须实现”的行为，子类继承后必须实现这些方法，起到类似接口的约束作用。

模板设计：可以定义一套调用流程的骨架方法（模板方法模式），具体的细节由子类实现。

说一说Java 中的static关键字

回答： 在 Java 中，static 关键字用于修饰类成员（字段、方法、内部类）和代码块，其核心作用是将成员与类本身关联，而非与类的实例关联。

static 修饰的成员属于类，而非类的某个实例

static 关键字通过将成员与类绑定，提供了全局共享和便捷访问的能力，但需谨慎使用以避免线程安全

深拷贝和浅拷贝的区别？

浅拷贝复制对象的引用，深拷贝复制对象的内容，即创建一个新对象，然后复制对象中的内容。

String、StringBuffer、StringBuilder 的区别？

String 是不可变类，StringBuffer 和 StringBuilder 是可变类，StringBuffer 是线程安全的， StringBuilder 是线程不安全的

throw 和 throws 的区别？

throw 用于抛出异常，throws 用于声明异常

什么是序列化和反序列化，序列化的条件是什么？

序列化： 将对象的状态转换为字节流的过程。

反序列化： 将字节流恢复为对象的过程。

条件： 类必须实现 Serializable 接口或者Jackson库序列化

不进行序列化后果：

  • 无法将对象保存到文件或数据库中
  • 无法在网络传输中传递对象，如 RPC（远程过程调用）中，客户端和服务器之间需要通过网络传输对象。
  • 在执行类似于对象持久化、远程通信、对象缓存等操作时，未序列化的对象将无法被处理，从而抛出运行时异常

常见的设计模式有哪些？

单例模式、工厂模式、观察者模式、策略模式、代理模式、装饰器模式、适配器模式等。

 * native 方法，用于返回当前运行时对象的 Class 对象，使用了 final 关键字修饰，故不允许子类重写。
 */
public final native Class<?> getClass()
/**
 * native 方法，用于返回对象的哈希码，主要使用在哈希表中，比如 JDK 中的HashMap。
 */
public native int hashCode()
/**
 * 用于比较 2 个对象的内存地址是否相等，String 类对该方法进行了重写以用于比较字符串的值是否相等。
 */
public boolean equals(Object obj)
/**
 * native 方法，用于创建并返回当前对象的一份拷贝。
 */
protected native Object clone() throws CloneNotSupportedException
/**
 * 返回类的名字实例的哈希码的 16 进制的字符串。建议 Object 所有的子类都重写这个方法。
 */
public String toString()
/**
 * native 方法，并且不能重写。唤醒一个在此对象监视器上等待的线程(监视器相当于就是锁的概念)。如果有多个线程在等待只会任意唤醒一个。
 */
public final native void notify()
/**
 * native 方法，并且不能重写。跟 notify 一样，唯一的区别就是会唤醒在此对象监视器上等待的所有线程，而不是一个线程。
 */
public final native void notifyAll()
/**
 * native方法，并且不能重写。暂停线程的执行。注意：sleep 方法没有释放锁，而 wait 方法释放了锁 ，timeout 是等待时间。
 */
public final native void wait(long timeout) throws InterruptedException
/**
 * 多了 nanos 参数，这个参数表示额外时间（以纳秒为单位，范围是 0-999999）。 所以超时的时间还需要加上 nanos 纳秒。。
 */
public final void wait(long timeout, int nanos) throws InterruptedException
/**
 * 跟之前的2个wait方法一样，只不过该方法一直等待，没有超时时间这个概念
 */
public final void wait() throws InterruptedException
/**
 * 实例被垃圾回收器回收的时候触发的操作
 * finalize() 方法用于在对象被垃圾回收前执行清理操作。但由于其行为不可控、
 * 性能问题以及可能导致资源泄漏，该方法已被 Java 官方弃用。
 */
protected void finalize() throws Throwable { }

说一说Java中的反射机制？

回答：

反射的定义

反射（Reflection） 是指在运行时动态获取类的信息（如类的结构、方法、字段等）并操作类的对象（如创建实例、调用方法、修改字段值等）的机制。

反射的实现原理

Java 反射的实现基于以下几个核心组件和机制：

1. 类加载机制与 Class 对象

  • 类加载过程：
    当 JVM 首次加载一个类时，会将该类的字节码（.class 文件）读入内存，并创建对应的 Class 对象。每个类在 JVM 中只有一个 Class 对象，它包含了该类的完整结构信息。

  • Class 对象的作用：
    Class 对象是反射的入口点，通过它可以获取类的所有信息（如方法、字段、构造函数等）。例如：

    Class<?> clazz = String.class;
Method[] methods = clazz.getMethods(); // 获取所有公共方法

2. 运行时类型信息（RTTI）

Java 在运行时为每个对象维护一个类型信息，通过 Class 对象和 Object.getClass() 方法可以获取该信息。反射利用这些信息在运行时动态解析和操作类的成员。

3. 反射 API 的核心类

Java 反射 API 主要位于 java.lang.reflect 包下，核心类包括：

  1. Class：表示类或接口，是反射的基础。
  2. Method：表示类的方法，可以动态调用。
  3. Field：表示类的字段，可以动态读写。
  4. Constructor：表示类的构造函数，可以动态创建对象。
  5. Modifier：提供访问修饰符（如 public、private）的信息。

示例：通过反射调用方法

Class<?> clazz = Class.forName("java.util.ArrayList");
Object instance = clazz.getDeclaredConstructor().newInstance(); // 创建 ArrayList 实例

Method addMethod = clazz.getMethod("add", Object.class); // 获取 add 方法
addMethod.invoke(instance, "hello"); // 调用 add 方法

4. 字节码解析与动态代理

  • 字节码解析：
    反射在运行时通过解析类的字节码（如方法签名、字段类型等）来获取类的结构信息。这需要 JVM 提供的底层支持。

  • 动态代理：
    反射可以在运行时生成代理类（如 Java 的 Proxy 类），实现 AOP（面向切面编程）等功能。动态代理通过字节码生成技术在运行时创建新类。

5. 访问权限控制

Java 反射允许通过 setAccessible(true) 打破访问限制，访问和修改私有成员：

Field privateField = MyClass.class.getDeclaredField("privateField");
privateField.setAccessible(true); // 打破 private 访问限制
privateField.set(instance, "new value");

反射的执行流程

  1. 获取 Class 对象：
    通过 Class.forName()、类名.class 或 对象.getClass() 获取目标类的 Class 对象。

  2. 获取成员对象：
    通过 Class 对象获取 Method、Field 或 Constructor 对象。

  3. 执行操作：

    • 调用 Method.invoke() 执行方法。
    • 调用 Field.get()/set() 读写字段值。
    • 调用 Constructor.newInstance() 创建对象。

反射的典型应用场景

  1. 框架开发（如 Spring、Hibernate）：动态创建对象、注入依赖。
  2. 序列化/反序列化（如 JSON 库）：动态读写对象字段。
  3. 测试工具：访问私有方法和字段进行单元测试。
  4. 插件系统：动态加载和使用外部插件。
