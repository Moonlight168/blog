List, Set, Queue, Map 有什么区别？

回答：

  • List(对付顺序的好帮手): 存储的元素是有序的、可重复的。
  • Set(注重独一无二的性质): 存储的元素不可重复的。
  • Queue(实现排队功能的叫号机): 按特定的排队规则来确定先后顺序，存储的元素是有序的、可重复的。
  • Map(用 key 来搜索的专家): 使用键值对（key-value）存储，类似于数学上的函数 y=f(x)，"x" 代表 key，"y" 代表 value，key 是无序的、不可重复的，value 是无序的、可重复的，每个键最多映射到一个值。集合框

ArrayList 和 数组 有什么区别？

ArrayList基于动态数组实现，相比静态数组Array更灵活：

  1. 容量可变：ArrayList可自动扩容缩容，Array长度固定；
  2. 类型安全：ArrayList支持泛型，Array不支持；
  3. 存储类型：ArrayList仅存对象（需包装基本类型），Array可存基本类型和对象；
  4. 操作便捷：ArrayList提供add、remove等丰富API，Array仅支持下标访问；
  5. 创建差异：ArrayList无需预设大小，Array创建时必须指定长度 。

为什么要使用集合，如何选择？

一、使用集合的原因

  1. 动态扩容：不用提前固定大小，能根据添加元素数量自动扩充容量。
  2. 功能丰富：自带很多方法，像添加、删除、查找等，用起来方便。
  3. 类型安全：通过泛型，规定集合里能放的数据类型，避免出错。
  4. 高性能：Java自带的集合都经过优化，处理数据速度快。

二、选择集合的方法

  1. 按数据特点选
    • 允许重复数据：选List。查找多就用ArrayList，增删多就用LinkedList。
    • 不允许重复数据：选Set。无序就用HashSet，要排序就用TreeSet 。
    • 存键值对数据：选Map。查找快用HashMap，按键排序用TreeMap。
  2. 按线程环境选
    • 单线程：用ArrayList、HashMap等非同步集合，速度快。
    • 多线程：用Vector、ConcurrentHashMap等同步集合，保证数据安全。

说一说ArrayList 的扩容机制

1. 初始容量与默认值

  • 初始容量：创建 ArrayList 时，若不指定大小（如 new ArrayList()），默认初始容量是 0（JDK 8+ 后）。
  • 第一次扩容：当首次添加元素时，容量会被扩展为 10。

2. 扩容触发条件

  • 核心逻辑：当 元素数量超过当前数组长度 时，触发扩容。
  • 示例：若当前容量为 10，添加第 11 个元素时会触发扩容。

3. 扩容步骤（源码解析）

假设原容量为 oldCapacity，扩容分为 3 步：

① 计算新容量

新容量 = oldCapacity + (oldCapacity >> 1)
即 原容量的 1.5 倍（向下取整）。
例如：原容量 10 → 新容量 10 + 5 = 15。

② 检查是否足够

若新容量仍小于所需最小容量（如添加第 16 个元素时，15 不够用），则直接扩容为 所需的最小容量。

③ 检查是否超出最大容量

  • 若新容量超过 Integer.MAX_VALUE - 8，会进一步调整为 Integer.MAX_VALUE。
  • 若超出 Integer.MAX_VALUE，抛出 OutOfMemoryError。

4. 数组复制

扩容后，ArrayList 会通过 Arrays.copyOf() 方法将原数组的元素复制到新数组中，这一步是 性能开销的主要来源。

5. 手动预分配容量

若提前知道数据量（如 1000 个元素），可通过 new ArrayList<>(1000) 指定初始容量，避免多次扩容，提升性能。

面试答题模板

面试官问：“ArrayList 如何扩容？”

回答：
1. 初始容量：默认 0（首次添加时扩容为 10）。
2. 触发条件：元素数量超过数组长度时。
3. 扩容公式：新容量 = 原容量 * 1.5（如 10 → 15 → 22 → 33 ...）。
4. 关键方法：通过 Arrays.copyOf() 复制数组，性能开销较大。
5. 优化建议：若数据量已知，可通过构造函数预分配容量，减少扩容次数。

源码验证（JDK 17）

private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1); // 1.5倍
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity; // 若1.5倍不够，直接用minCapacity
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity); // 处理超大容量
    elementData = Arrays.copyOf(elementData, newCapacity); // 复制数组
}

说一说Comparable 和 Comparator 的区别？

回答：

Comparable 和 Comparator 是 Java 中用于对象排序的两个核心接口，它们的区别可以用「自带排序」和「外部策略」来概括：

1. 核心区别

维度ComparableComparator
接口定义java.lang.Comparablejava.util.Comparator
排序逻辑类自身实现 compareTo() 方法，定义「自然排序」规则。在类外部实现 compare() 方法，定义「临时排序」规则。
耦合度排序逻辑与类绑定，侵入性强。排序逻辑与类分离，解耦性好。
使用场景类的默认排序方式（如 String 按字典序）。类已有默认排序，但需要额外排序规则（如逆序、按年龄）。

2. 代码对比

Comparable 示例（自带排序）

// 类实现 Comparable 接口
public class Person implements Comparable<Person> {
    private String name;
    private int age;

    @Override
    public int compareTo(Person other) {
        return this.age - other.age; // 按年龄升序
    }
}

// 使用：直接调用 Collections.sort()
List<Person> people = new ArrayList<>();
Collections.sort(people); // 自动按 Person 的 compareTo() 排序

Comparator 示例（外部策略）

// 类无需实现任何接口
public class Person {
    private String name;
    private int age;
}

// 定义外部 Comparator
Comparator<Person> ageComparator = new Comparator<Person>() {
    @Override
    public int compare(Person p1, Person p2) {
        return p1.getAge() - p2.getAge(); // 按年龄升序
    }
};

// 使用：在排序时传入 Comparator
Collections.sort(people, ageComparator); // 按指定规则排序

3. 关键方法

  • Comparable：
    int compareTo(T o)
    返回值规则：

    • this < o → 返回负数（如 -1）
    • this == o → 返回 0
    • this > o → 返回正数（如 1）

  • Comparator：
    int compare(T o1, T o2)
    返回值规则与 compareTo 一致，但比较的是两个参数。

4. 优缺点

接口优点缺点
Comparable实现后可直接排序，使用简单。排序规则固定，无法灵活切换。
Comparator支持多维度排序，不修改原类。需要额外创建 Comparator 实例。

5. 常见应用场景

  • Comparable：

    • String、Integer 等类默认实现 Comparable。
    • 自定义实体类（如 User、Product）的默认排序。

  • Comparator：

    • 集合临时排序（如 Arrays.sort(arr, comparator)）。
    • GUI 中的表格排序（可切换升序/降序）。
    • 第三方库的排序策略（如 Stream.sorted(comparator)）。

6. 面试高频问题

问：“如何对一个没有实现 Comparable 的类排序？”
答：

  1. 不修改原类：创建外部 Comparator 实现排序规则。
  2. 修改原类：让类实现 Comparable 接口，添加 compareTo() 方法。

总结

  • 用 Comparable：当类需要有一个固定的、默认的排序方式时。
  • 用 Comparator：当需要灵活切换排序规则或对未实现 Comparable 的类排序时。

为什么泛型不能直接使用基本类型？那如何在泛型中使用基本类型？

回答：

泛型在编译时会进行类型擦除，最终所有泛型类型通常都会被替换为 Object，而基本类型不是 Object 的子类，因此无法直接作为类型实参。

使用对应的包装类（如 Integer、Double 等），Java 会自动进行装箱和拆箱操作。

Java中的类型擦除是什么？这样做的目的和后果是什么？

回答：

1. 类型擦除的定义

  • 编译阶段：Java编译器会将所有泛型类型参数（如List<Integer>中的Integer）替换为其上限类型（通常是Object，若有显式边界则为边界类型）。
  • 运行阶段：泛型类型不再包含实际的类型参数信息，例如List<Integer>和List<String>在运行时是相同的类型（List）。

2. 擦除规则示例

① 无边界的泛型类型 → 擦除为Object

class Box<T> {
    private T value; // 编译后：private Object value;
    public T get() { return value; } // 编译后：public Object get()
}

② 有显式边界的泛型类型 → 擦除为边界类型

class Box<T extends Number> { // 边界为Number
    private T value; // 编译后：private Number value;
}

3. 类型擦除的目的

  1. 保持二进制兼容性：
    Java 5引入泛型时，为了不破坏现有代码（如List仍可与List<Object>兼容），选择在编译阶段移除泛型信息。

  2. 简化实现：
    避免为每种泛型类型生成额外的类（如List<Integer>和List<String>无需单独编译），减少字节码体积。

4. 类型擦除的后果

① 运行时无法获取泛型的具体类型参数

List<Integer> intList = new ArrayList<>();
List<String> strList = new ArrayList<>();

System.out.println(intList.getClass() == strList.getClass()); // true
// 运行时两者都是ArrayList，类型参数被擦除

② 不能创建泛型数组

List<Integer>[] array = new List<Integer>[10]; // 编译错误！
// 原因：类型擦除后无法验证数组元素类型（可能存入List<String>）

③ 泛型类型不能是基本类型

List<int> list = new ArrayList<>(); // 错误！
// 类型擦除后需存储Object，但基本类型（如int）不是Object的子类

④ 静态成员不能引用类的泛型参数

class Box<T> {
    private static T value; // 编译错误！静态成员无法使用泛型类型参数
}

5. 桥接方法（Bridge Method）

为了支持泛型的多态性，编译器会生成桥接方法。例如：

class StringBox extends Box<String> {
    @Override
    public String get() { return super.get(); }
}

编译后会生成两个方法：

public String get() { ... } // 用户定义的方法
public Object get() { return get(); } // 编译器生成的桥接方法，覆盖Box<Object>的get()

6. 面试高频回答

Q1：“为什么Java选择类型擦除？”
A1：为了在引入泛型时保持与旧版本代码的二进制兼容性，避免修改现有类库。

Q2：“类型擦除会导致哪些问题？”
A2：

  • 运行时无法区分List<Integer>和List<String>。
  • 不能创建泛型数组（如List<Integer>[]）。
  • 泛型类型参数不能是基本类型（需用包装类）。

Q3：“如何绕过类型擦除限制？”
A3：

  • 通过反射获取泛型信息（如getGenericSuperclass()）。
  • 使用通配符（如List<?>）结合反射操作。
