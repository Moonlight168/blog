Spring、SpringBoot 的常用注解？

回答：

1. 核心容器注解

  1. 组件扫描与 Bean 定义

    • @Component：通用组件注解，标记类为 Spring Bean
    • @Repository：DAO 层组件，提供数据访问异常转换
    • @Service：服务层组件，标记业务逻辑类
    • @Controller：控制器层组件，标记 Web 控制器
    • @Configuration：配置类，替代 XML 配置文件
    • @Bean：声明 Bean 方法，通常在 @Configuration 类中使用
    • @ComponentScan：指定扫描路径，自动发现组件
    • @Import：导入其他配置类或组件

  2. 依赖注入

    • @Autowired：按类型自动注入依赖
    • @Qualifier：与 @Autowired 配合，按名称指定注入 Bean
    • @Resource：JSR-250 标准注解，按名称或类型注入
    • @Value：注入配置文件中的值（如 application.properties）
    • @Required：标记 Bean 属性为必需，初始化时检查

  3. 作用域与生命周期

    • @Scope：指定 Bean 的作用域（如 singleton, prototype）
    • @Lazy：延迟初始化 Bean
    • @PostConstruct：Bean 初始化后执行的方法
    • @PreDestroy：Bean 销毁前执行的方法

2. AOP 注解

  1. 切面定义

    • @Aspect：标记切面类
    • @Pointcut：定义切入点表达式
    • @Order：指定切面执行顺序

  2. 通知类型

    • @Before：前置通知
    • @After：后置通知（无论是否异常）
    • @AfterReturning：返回通知（方法正常返回后）
    • @AfterThrowing：异常通知（方法抛出异常时）
    • @Around：环绕通知，可控制方法执行

3. 事务管理注解

  • @Transactional：声明事务属性，支持隔离级别、传播行为等配置
  • @EnableTransactionManagement：启用声明式事务支持

4. Spring MVC 注解

  1. 控制器与请求处理

    • @RestController：组合 @Controller 和 @ResponseBody，返回 JSON/XML
    • @RequestMapping：映射请求路径和 HTTP 方法
    • @GetMapping、@PostMapping 等：简化的请求映射注解
    • @RequestBody：将请求体解析为对象
    • @ResponseBody：将返回值直接作为响应体
    • @PathVariable：获取路径变量
    • @RequestParam：获取请求参数
    • @RequestHeader：获取请求头
    • @CookieValue：获取 Cookie 值
    • @ModelAttribute：将请求参数绑定到模型
    • @SessionAttributes：将模型属性存储在会话中

  2. 异常处理

    • @ExceptionHandler：处理控制器内的异常
    • @ControllerAdvice：全局异常处理类
    • @RestControllerAdvice：组合 @ControllerAdvice 和 @ResponseBody

Spring Boot 常用注解

1. 启动与自动配置

  • @SpringBootApplication：组合 @Configuration、@EnableAutoConfiguration 和 @ComponentScan，标记主应用类
  • @EnableAutoConfiguration：启用 Spring Boot 的自动配置机制
  • @SpringBootTest：Spring Boot 集成测试注解
  • @AutoConfigureMockMvc：自动配置 MockMvc 进行 Web 测试

2. 配置属性绑定

  • @ConfigurationProperties：将配置文件属性绑定到 Bean
  • @EnableConfigurationProperties：启用 @ConfigurationProperties 注解的 Bean
  • @PropertySource：加载自定义属性文件
  • @Value：获取单个配置值

3. 其他增强注解

  • @WebMvcTest：只测试 Spring MVC 组件，不启动完整应用
  • @DataJpaTest：只测试 JPA 组件，不启动完整应用
  • @TestConfiguration：测试专用配置类
  • @MockBean：在测试中替换 Bean 为 Mock 对象
  • @TestPropertySource：测试环境专用配置
  • @RefreshScope：配置刷新时重新创建 Bean（用于配置中心）

什么是 Spring？它有哪些核心模块？

回答： Spring 是一个轻量级的 Java 开发框架，核心模块包括：

  • Spring Core（IOC 容器）
  • Spring AOP（面向切面编程）
  • Spring MVC（Web 框架）
  • Spring Data、Spring Security 等

什么是 IOC 和 AOP？

回答：

  • IOC（控制反转）：对象创建与依赖注入由容器管理，降低耦合。
  • AOP（面向切面编程）：用来处理日志、事务等横切逻辑。

AOP的常用的注解有哪些？实现原理？

回答：

1. 常用注解

Spring AOP 基于 AspectJ 注解提供了强大的面向切面编程支持，常用注解包括：

  1. @Aspect
    声明一个类为切面类，需配合 @Component 注解使用。

    @Aspect
@Component
public class LoggingAspect {
    // 切面逻辑
}

  2. @Pointcut
    定义切入点表达式，指定在哪些方法执行时触发通知。

    @Pointcut("execution(* com.example.service.*.*(..))")
public void serviceMethods() {} // 切入点签名

  3. @Before
    前置通知：在目标方法执行前执行。

    @Before("serviceMethods()")
public void beforeAdvice(JoinPoint joinPoint) {
    System.out.println("Before method: " + joinPoint.getSignature().getName());
}

  4. @After
    后置通知：无论目标方法是否抛出异常，都会执行。

    @After("serviceMethods()")
public void afterAdvice() {
    System.out.println("After method execution");
}

  5. @AfterReturning
    返回通知：在目标方法正常返回后执行，可获取返回值。

    @AfterReturning(pointcut = "serviceMethods()", returning = "result")
public void afterReturningAdvice(Object result) {
    System.out.println("Method returned: " + result);
}

  6. @AfterThrowing
    异常通知：在目标方法抛出异常时执行。

    @AfterThrowing(pointcut = "serviceMethods()", throwing = "ex")
public void afterThrowingAdvice(Exception ex) {
    System.out.println("Exception caught: " + ex.getMessage());
}

  7. @Around
    环绕通知：环绕目标方法执行，可控制方法的执行时机和结果。

    @Around("serviceMethods()")
public Object aroundAdvice(ProceedingJoinPoint joinPoint) throws Throwable {
    System.out.println("Before method proceed");
    Object result = joinPoint.proceed(); // 执行目标方法
    System.out.println("After method proceed");
    return result;
}

  8. @Order
    指定切面执行顺序，值越小优先级越高。

    @Aspect
@Component
@Order(1) // 优先级高于 @Order(2) 的切面
public class TransactionAspect {
    // ...
}

2. 实现原理

Spring AOP 基于代理模式实现，主要有两种代理方式：

  1. JDK 动态代理
    基于接口的代理，代理类实现目标接口。

    // 示例接口
public interface UserService {
    void createUser(String username);
}

// JDK 动态代理生成的代理类（简化）
public class $Proxy0 implements UserService {
    private UserService target;
    
    public $Proxy0(UserService target) {
        this.target = target;
    }
    
    @Override
    public void createUser(String username) {
        // 前置通知
        beforeAdvice();
        try {
            target.createUser(username); // 执行目标方法
            // 返回通知
            afterReturningAdvice();
        } catch (Exception e) {
            // 异常通知
            afterThrowingAdvice(e);
        } finally {
            // 后置通知
            afterAdvice();
        }
    }
}

  2. CGLIB 代理
    基于子类的代理，通过字节码生成技术创建目标类的子类。

    // 示例类（无接口）
public class UserServiceImpl {
    public void createUser(String username) {
        // ...
    }
}

// CGLIB 生成的子类（简化）
public class UserServiceImpl$$EnhancerByCGLIB$$ extends UserServiceImpl {
    private MethodInterceptor interceptor;
    
    @Override
    public void createUser(String username) {
        // 调用拦截器，执行通知逻辑
        interceptor.intercept(this, method, args, methodProxy);
    }
}

3. AOP 代理创建流程

  1. 解析切面配置
    Spring 在启动时扫描带有 @Aspect 注解的类，解析其中的通知（Advice）和切入点（Pointcut）。

  2. 创建 BeanPostProcessor
    AnnotationAwareAspectJAutoProxyCreator 作为后置处理器，在 Bean 初始化后检查是否需要创建代理。

  3. 匹配切入点
    对每个 Bean 的方法进行切入点表达式匹配，若匹配则创建代理。

  4. 生成代理对象

    • 若目标类实现了接口，使用 JDK 动态代理。
    • 若目标类未实现接口（或 proxy-target-class=true），使用 CGLIB 代理。

  5. 代理链调用
    代理对象执行方法时，按顺序调用所有匹配的通知，最后调用目标方法。

4. 关键组件

  1. ProxyFactory
    创建代理对象的工厂类，根据配置选择 JDK 或 CGLIB 代理。

  2. Advisor
    包含一个通知（Advice）和一个切入点（Pointcut），定义何时何地执行通知。

  3. MethodInterceptor
    拦截器接口，环绕通知通过实现该接口来控制方法的执行。

  4. JoinPoint
    连接点，表示程序执行过程中的某个点（如方法调用），通知可通过它获取上下文信息。

5. AOP vs AspectJ

Spring AOP 是基于代理的轻量级实现，而 AspectJ 是完整的 AOP 框架，支持：

  • 编译时织入（Compile-time weaving）
  • 加载时织入（Load-time weaving）
  • 运行时织入（类似 Spring AOP）

Spring AOP 仅支持运行时织入，而 AspectJ 功能更强大，但配置更复杂。

Spring 是如何实现依赖注入的？底层是如何实现的？

回答：

Spring 的依赖注入（Dependency Injection, DI） 是其核心特性之一，通过反射和 IoC（控制反转）容器实现对象间的松耦合。

1. 依赖注入的基本概念

  • 依赖注入：对象的依赖由外部容器动态提供，对象仅通过接口或抽象类引用依赖。

示例：

public class UserController {
    private UserService userService; // 依赖声明
    
    // 通过构造函数注入依赖（Constructor Injection）
    public UserController(UserService userService) {
        this.userService = userService;
    }
}

2. Spring 依赖注入的底层实现

① BeanDefinition 与 Bean 元数据

  • BeanDefinition：Spring 容器在启动时解析配置（XML、注解或 Java 配置），将每个 Bean 的定义信息（如类名、作用域、依赖关系）封装为 BeanDefinition 对象。
  • 注册表：BeanDefinitionRegistry 存储所有 BeanDefinition，作为创建 Bean 的蓝图。

流程：

// 伪代码：Spring 解析 Bean 定义
BeanDefinition bd = new RootBeanDefinition(UserServiceImpl.class);
bd.setScope("singleton");
registry.registerBeanDefinition("userService", bd);

② BeanFactory 与反射创建对象

  • BeanFactory：Spring 的核心工厂接口，负责创建和管理 Bean。
  • 反射实例化：通过 Java 反射调用构造函数创建 Bean 实例。

示例：

// 伪代码：Spring 通过反射创建 Bean
Class<?> beanClass = Class.forName(beanDefinition.getClassName());
Constructor<?> constructor = beanClass.getDeclaredConstructor();
Object beanInstance = constructor.newInstance(); // 反射创建实例

③ 依赖解析与注入

  • 依赖查找：根据 BeanDefinition 中的依赖描述，从容器中查找匹配的 Bean。
  • 反射注入：通过反射设置字段值（Field Injection）或调用 setter 方法（Setter Injection）。

示例：

// 伪代码：Spring 通过反射注入依赖
Field field = beanClass.getDeclaredField("userService");
field.setAccessible(true); // 打破 private 限制
field.set(beanInstance, userServiceBean); // 注入依赖

④ 后置处理器（BeanPostProcessor）

  • 增强 Bean：允许在 Bean 实例化后、初始化前后进行自定义处理（如 AOP 代理）。
  • 常见实现：AutowiredAnnotationBeanPostProcessor 处理 @Autowired 注解。

示例：

// 伪代码：处理 @Autowired 注解
if (annotation instanceof Autowired) {
    // 查找依赖 Bean
    Object dependency = beanFactory.getBean(dependencyType);
    // 通过反射注入
    field.set(bean, dependency);
}

3. 注解驱动的依赖注入（@Autowired）

当使用 @Autowired 注解时，Spring 的处理流程：

  1. 扫描注解：
    ComponentScan 扫描带有 @Component、@Service 等注解的类，注册为 BeanDefinition。

  2. 解析依赖：
    AutowiredAnnotationBeanPostProcessor 在 Bean 实例化后，检查字段或方法上的 @Autowired 注解。

  3. 依赖匹配：
    根据类型（byType）或名称（byName，结合 @Qualifier）从容器中查找匹配的 Bean。

  4. 反射注入：
    通过反射设置依赖值。

4. 构造函数注入 vs. Setter 注入

  • 构造函数注入：
    通过构造函数提供依赖，确保对象创建后即处于完整状态，推荐用于必需依赖。

    public UserController(UserService userService) {
    this.userService = userService;
}

  • Setter 注入：
    通过 setter 方法提供依赖，适用于可选依赖。

    @Autowired
public void setUserService(UserService userService) {
    this.userService = userService;
}

5. Spring 5+ 的改进：构造函数注入优先

从 Spring 5 开始，推荐使用无 @Autowired 注解的构造函数注入：

public UserController(UserService userService) { // 无需 @Autowired
    this.userService = userService;
}
  • 优势：
    1. 隐式标记必需依赖。
    2. 支持不可变对象（final 字段）。
    3. 防止循环依赖（编译时错误）。

6. 循环依赖处理

当 Bean A 依赖 Bean B，而 B 又依赖 A 时，Spring 通过三级缓存解决：

  1. 一级缓存（singletonObjects）：存储完全初始化的单例 Bean。
  2. 二级缓存（singletonFactories）：存储早期暴露的 Bean 工厂（用于 AOP 代理）。
  3. 三级缓存（earlySingletonObjects）：存储提前创建的原始 Bean 实例。

处理流程：

  1. 创建 A 实例，将 A 的工厂放入三级缓存。
  2. 解析 A 的依赖 B，发现 B 未创建，开始创建 B。
  3. 创建 B 实例，将 B 的工厂放入三级缓存。
  4. 解析 B 的依赖 A，从三级缓存获取 A 的工厂，生成 A 的早期引用注入到 B。
  5. B 完成初始化，放入一级缓存。
  6. A 获取 B 的引用，完成初始化，放入一级缓存。

第三方的bean如何交给spring去管理？

回答：

一、注册第三方 Bean 的方式

  1. @Bean 注解（最常用）

    • 在 @Configuration 类中定义方法返回第三方组件实例。
    • 示例：
      @Configuration  
public class AppConfig {  
    @Bean  
    public RestTemplate restTemplate() {  
        return new RestTemplate();  
    }  
}

  2. @Import 导入配置类

    • 若第三方提供配置类，直接导入。
    • 示例：
      @Import(ThirdPartyConfig.class)  
public class AppConfig {}

  3. FactoryBean 接口

    • 用于复杂组件的创建逻辑。
    • 示例：
      public class MyFactoryBean implements FactoryBean<ThirdPartyClass> {  
    // 实现接口方法  
}

  4. 自动配置（Spring Boot）

    • 使用 @ConditionalOnMissingBean 提供默认实现。
    • 示例：
      @Bean  
@ConditionalOnMissingBean  
public ThirdPartyClass thirdPartyClass() {  
    return new ThirdPartyClass();  
}

  5. XML 配置（兼容旧系统）

    • 通过 @ImportResource 导入 XML 文件。
    • 示例：
      @ImportResource("classpath:thirdparty.xml")

二、使用注册后的 Bean

  1. 依赖注入（推荐）

    • 通过构造函数或字段注入。
    • 示例：
      @Service  
public class MyService {  
    private final RestTemplate restTemplate;  

    public MyService(RestTemplate restTemplate) {  
        this.restTemplate = restTemplate;  
    }  
}

  2. 通过 ApplicationContext 获取

    • 在非 Spring 管理的类中手动获取。
    • 示例：
      ApplicationContext context = ...;  
RestTemplate restTemplate = context.getBean(RestTemplate.class);

  3. 在配置类内部使用

    • 直接调用 @Bean 方法（Spring 确保返回单例）。
    • 示例：
      @Configuration  
public class AppConfig {  
    @Bean  
    public MyService myService() {  
        return new MyService(restTemplate());  
    }  
}

三、关键区别

场景推荐方式原因
自有类@Component直接标记类，简单高效
第三方类@Bean无需修改源码，灵活配置
复杂创建逻辑FactoryBean封装复杂初始化过程
Spring Boot@ConditionalOnMissingBean自动配置，避免重复定义

四、注意事项

  1. 单例作用域：默认 @Bean 创建的是单例 Bean，可通过 @Scope 修改。
  2. 依赖顺序：@Bean 方法可通过参数注入依赖，Spring 自动处理顺序。
  3. 避免循环依赖：构造函数注入时需注意依赖关系，优先使用 setter 注入。
