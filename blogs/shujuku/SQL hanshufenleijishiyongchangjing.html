SQL 函数分类及使用场景

一、单行函数（Scalar Functions）

定义：对每一行数据独立处理，每行输入返回一个结果，与其他行无关。
特点：输入一行，输出一行，不依赖其他行的数据。
常见分类及示例：

1. 字符串函数（String Functions）
  • 作用：处理字符串数据（拼接、截取、转换大小写、计算长度等）。
    • CONCAT(str1, str2)：拼接字符串（MySQL、SQL Server）。
      SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM users;
    • SUBSTRING(str, start, length)：截取子字符串（MySQL）。
      SELECT SUBSTRING(email, 1, 5) AS short_email FROM users;
    • UPPER(str) / LOWER(str)：转换大小写。
    • LENGTH(str) / CHAR_LENGTH(str)：计算字节长度/字符数（MySQL）。
    • TRIM() / LTRIM() / RTRIM()：去除首尾/左/右空格。
2. 数值函数（Numeric Functions）
  • 作用：处理数值数据（数学运算、取整、随机数等）。
    • ABS(num)：绝对值。
    • ROUND(num, decimals)：四舍五入。
      SELECT ROUND(price, 2) AS rounded_price FROM products;
    • FLOOR(num) / CEILING(num)：向下取整/向上取整。
    • RAND()：生成随机数（MySQL）。
3. 日期和时间函数（Date/Time Functions）
  • 作用：处理日期和时间数据（提取部分字段、计算间隔、格式化等）。
    • CURDATE() / CURTIME()：获取当前日期/时间（MySQL）。
    • DATE_FORMAT(date, format)：格式化日期。
      SELECT DATE_FORMAT(order_date, '%Y-%m-%d') AS formatted_date FROM orders;
    • DATEDIFF(date1, date2)：计算两个日期的天数差。
      SELECT DATEDIFF(return_date, order_date) AS days_diff FROM orders;
4. 类型转换函数（Conversion Functions）
  • 作用：转换数据类型（如字符串转数值、日期转字符串等）。
    • CAST(expr AS type)：显式类型转换。
      SELECT CAST('123' AS SIGNED) AS num FROM dual; -- MySQL 中转为整数
    • CONVERT(expr, type)：另一种转换方式（不同数据库语法可能不同）。
5. 条件函数（Conditional Functions）
  • 作用：根据条件返回不同结果。
    • IF(condition, value_if_true, value_if_false)：简单条件判断（MySQL）。
      SELECT price, IF(price > 100, '高价', '低价') AS price_level FROM products;
    • CASE WHEN THEN ELSE END：多条件判断。
      SELECT 
  score,
  CASE 
    WHEN score >= 90 THEN '优秀'
    WHEN score >= 80 THEN '良好'
    ELSE '一般'
  END AS grade
FROM students;

使用场景：

  • 在 SELECT 子句中直接处理字段值（如格式化、计算）。
  • 在 WHERE 子句中过滤数据（如 WHERE LENGTH(name) > 5）。
  • 在 ORDER BY、GROUP BY 子句中配合使用（如按字符串长度排序）。

二、聚合函数（Aggregate Functions）

定义：对一组数据进行统计计算，返回一个汇总结果（多行输入，返回一行）。
特点：自动忽略 NULL 值（除 COUNT(*)），常与 GROUP BY 子句配合使用。

常见聚合函数及示例：

函数作用示例
COUNT(expr)统计非 NULL 值的行数SELECT COUNT(age) FROM users;（忽略 age 为 NULL 的行）
COUNT(*)统计所有行数（包括 NULL）SELECT COUNT(*) FROM users;
SUM(expr)求和SELECT SUM(price) AS total_price FROM orders;
AVG(expr)求平均值SELECT AVG(score) AS avg_score FROM students;
MAX(expr)求最大值SELECT MAX(salary) AS max_salary FROM employees;
MIN(expr)求最小值SELECT MIN(stock_quantity) AS min_stock FROM products;
GROUP_CONCAT(expr)分组拼接字符串（MySQL）SELECT department, GROUP_CONCAT(name SEPARATOR ',') AS members FROM employees GROUP BY department;

使用限制：

  • 不能直接在 WHERE 子句中使用聚合函数，因为 WHERE 执行时数据尚未分组，聚合函数需基于分组后的数据计算。
    • 错误示例：SELECT * FROM orders WHERE SUM(price) > 1000;（报错）。
  • 若需过滤聚合结果，需使用 HAVING 子句（HAVING 在分组后执行）。
    SELECT category, SUM(price) AS total_price
FROM products
GROUP BY category
HAVING total_price > 1000; -- 过滤分组后的聚合结果

使用场景：

  • 统计总数、平均值、总和等业务指标（如“各部门员工平均工资”）。
  • 与 GROUP BY 配合，按字段分组统计（如“按月份统计订单量”）。
  • 与 DISTINCT 配合（如 COUNT(DISTINCT user_id) 统计唯一用户数）。

三、其他函数分类

1. 窗口函数（Window Functions）
  • 作用：对分组数据进行排名、累计计算等，不改变行数（保留原始行数据）。
    • 示例：RANK()、DENSE_RANK()、SUM(expr) OVER (PARTITION BY col ORDER BY col)。
    SELECT 
  employee_name, 
  salary, 
  RANK() OVER (ORDER BY salary DESC) AS salary_rank
FROM employees;
  • 使用场景：报表中的排名、同组内对比（如“各部门员工薪资排名”）。
2. 系统函数（System Functions）
  • 作用：获取数据库系统信息（用户、版本、当前时间等）。
    • 示例：USER()（获取当前用户，MySQL）、VERSION()（获取数据库版本）。
